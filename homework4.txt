1. What is .gitignore? You can write the answer either in Korean or English.

해당 git에서 가장 상위 디렉토리에 존재할 때, 이것을 포함해 push를 해주면 .gitignore의 내용에 따라 git에서 몇몇 파일들이 제외된다.
즉, 유저가 .gitignore의 내용을 임의로 입력하여, 해당 프로젝트에서, 로컬에 있는 원하지 않는 파일들을 지정하여, 이후 리모트에 push해 올릴 때 git에서 제외시킬 수 있다는 것이다.
그 대략적인 문법은 아래와 같다 :
#이 있는 줄은 주석처리가 된다.
원하지 않는 확장자명은 *.확장자명 을 통해 지정해줄 수 있다.
앞서 제외한 확장자명 중 예외로 할 파일은 !파일이름.확장자명 을 통해 지정해줄 수 있다.
현재 디렉토리의 특정 파일만을 제외하고 싶다면 /파일명 을 해주면 된다.
어떤 디렉토리의 내용을 전부 제외하고 싶다면 디렉토리명/ 을 해주면 된다.
해당 디렉토리의 (단, 서브디렉토리는 제외하고) 특정 확장자명의 모든 파일을 제외하고 싶다면 디렉토리명/*.확장자명 을 해주면 된다.
해당 디렉토리의 서브디렉토리까지 포함하여 특정 확장자명의 모든 파일을 제외하고 싶다면 디렉토리명/**/*.확장자명 을 해주면 된다.


2. Why do Github users need an SSH key pair? (Users can use either SSH key pair or github account info.) Please write down a brief explanation of SSH key.

작업물을 보호하기 위해, 자신의(혹은 그렇지 않더라도) github의 repository에 작업물을 push할 수 있는 사람이라는 것을 
인증하기 위하여 SSH key 가 필요하게 되는데, 이는 서버(인스턴스)의 주인 본인뿐만 아니라 그가 원하는 지정된 사람들도 그것에
연결할 권한을 주는 것이다. SSH 프로토콜을 이용한 것으로, 이를 이용하면 매번 유저명과 비밀번호를 입력하지 않고 쉽게 
연결하는 것 또한 가능하다. 개인키와 공개키로 이루어져 있는데, 개인키는 본인의 디바이스에 보관되고, 공개키는 원하는 대로 
배포하여 공개키를 가지게 된 사람들 또한 그것으로 인증하여 그 서버에 접근할 수 있도록 해 준다.
